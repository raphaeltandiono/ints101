Analysis Plan

Quantitative Analysis: Regression Modeling

Using regression models (e.g., panel regressions) to test the relationship between network structure and resilience indicators across time and ecosystems.

Model Concept:

Resilience Indicatore,t = β0 + β1(Network Metrice,t-1) + β2(Downturn Dummyt) + β3(Network Metrice,t-1 * Downturn Dummyt) + Controlse,t + εe,t

Where 'e' is ecosystem, 't' is time period. The interaction term (β3) is key to testing how network structure moderates or correlates with resilience *during* a downturn.

Addressing Endogeneity:

Using lagged independent variables (e.g., network structure *before* a downturn to predict resilience *during* or *after* it). Potential use of Instrumental Variables if applicable.

Hypotheses Testing:

Statistical tests on coefficients (especially β1 and β3) related to specific hypotheses (H1-H3, as mentioned in the plan) about the effects of density, bridging capital, etc., on different resilience dimensions.

Qualitative Analysis: Thematic Analysis & Triangulation

Analyzing interview transcripts and case study data using thematic analysis to identify recurring patterns, challenges, adaptation strategies, and perceived roles of network relationships during crises.

Step 1: Coding Qualitative Data

Identify key themes related to network behavior, crisis impact, decision-making, and resilience factors from interviews.

Step 2: Identifying Underlying Mechanisms & Context

Explore the 'why' behind the quantitative findings. For example, if density is found to be protective, qualitative data might reveal that dense networks facilitate faster information flow about distressed companies or available follow-on funding.

Step 3: Triangulation

Comparing and integrating qualitative themes with quantitative results to strengthen causal inference and provide a richer, more nuanced understanding of the complex relationship between network dynamics and ecosystem resilience.
